# Client API Path Structures

This document provides the complete API path structure for each client endpoint. Use these paths when making requests to retrieve files.

**Request Format:**
```
POST https://rac-retrieval.vercel.app/api/rac/{client}
Content-Type: application/json

{
  "path": "LAYER/FILE_ID"
}
```

## Request Examples

### Basic File Request
```bash
curl -X POST "https://rac-retrieval.vercel.app/api/rac/msn" \
  -H "Content-Type: application/json" \
  -d '{"path": "ORG/FOUNDATION"}'
```

### Request Specific Section
```bash
curl -X POST "https://rac-retrieval.vercel.app/api/rac/fixico" \
  -H "Content-Type: application/json" \
  -d '{"path": "ORG/BRAND_POSITIONING/brand_statements"}'
```

### Request LIBRARY File
```bash
curl -X POST "https://rac-retrieval.vercel.app/api/rac/reducate" \
  -H "Content-Type: application/json" \
  -d '{"path": "LIBRARY/CME_ONLINE_BRAND"}'
```

### Request LOGIC File
```bash
curl -X POST "https://rac-retrieval.vercel.app/api/rac/aaa" \
  -H "Content-Type: application/json" \
  -d '{"path": "LOGIC/QUALITY_GATES"}'
```
